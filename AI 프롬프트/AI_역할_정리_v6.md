# AI 역할 정리 v4 (프로젝트 설계 기준)

## 1. 데이터 입력 시 — Free-text 분석 및 분배

### 기본 흐름
```
의사가 free-text로 차트 작성 (기존 습관 그대로)
    ↓
AI가 한 번의 API 호출로 두 가지를 동시 수행:
  ① 데이터 추출: free-text에서 정형 데이터 분류
  ② 차트 검토: 누락/오류/추가 제안
    ↓
각 테이블로 분배:
  - 진단 → 진단 테이블
  - 검사 수치 → 검사결과 테이블
  - 영상/특수검사 → 영상검사 테이블
  - 추적 계획 → 추적계획 테이블
  - 활력징후/생활습관 → 방문 테이블
  - 원본 free-text → 방문 테이블에 그대로 보존
    ↓
의사에게 추출 데이터 + 검토 의견을 한꺼번에 보여주기
    ↓
동의/반려 → 저장 (API 1번, 확인 1번, 저장 1번)
```

### AI 분석 실패 시 대응
```
AI 분석 실패 (API 오류, 서버 장애 등):
  → free_text만 방문 테이블에 저장 (분석완료=0)
  → 안내: "입력하신 차트는 free-text로 먼저 저장되었습니다.
           다음 접속 시 AI가 자동으로 분석을 시도합니다."

다음 로그인 시:
  → 프로그램 시작 시 분석완료=0인 차트 자동 체크
  → "미분석 차트 N건이 있습니다. 지금 분석할까요? (y/n)"
  → y: AI 분석 → 확인 → 저장 → 분석완료=1
  → n: 다음 접속 시 다시 물어봄
```

### 차트 스타일 인식 규칙
```
의사의 차트 작성 스타일:

1. 섹션 구분 — * (별표 1개)
   [섹션1: 진단/문제목록/검사결과]
   #진단명, 검사 수치 등
   *
   [섹션2: 신체검진/생활습관]
   BP, 키, 몸무게, 흡연, 음주, 운동 등
   *
   [섹션3: 진료내용]
   진료 경과, 설명, 문진 등 (free-text)
   *
   [섹션4: 계획]
   **약 처방, **추적검사 등

2. 별표 규칙:
   * (1개)  = 섹션 구분자
   ** (2개) = 계획 항목. 날짜 유무로 '이미 한 것'과 '앞으로 할 것'을 구분:

   [날짜 없는 **] → 방문일에 시행/처방한 것
   - **약 N개월 → 처방요약: 'N개월치 처방' + 추적계획: 방문일+N개월 후 재방문(YYMM)
   - **Lab(검사명) → 검사결과: 시행일=방문일(YYMMDD), 결과=미확인
   - **영상검사명 → 영상검사: 시행일=방문일(YYMMDD), 결과=미확인
   → 결과 확인은 다음 방문 또는 중간 방문에서

   [날짜 있는 **] → 해당 월에 할 예정 (변경 가능한 계획)
   - **YYMM Lab(검사명) → 추적계획: 예정일=YYMM(4자리 그대로)
   - **YYMM 검사명 → 추적계획: 예정일=YYMM(4자리 그대로)

   예시:
   **약 2개월              → 처방요약 + 추적계획(재방문, YYMM)
   **Lab(A1c, Lipid)       → 검사결과: 시행일=방문일, 결과=미확인
   **경동맥 초음파          → 영상검사: 시행일=방문일, 결과=미확인
   **2604 Lab(A1c, Lipid)  → 추적계획: 예정일=2604
   **2701 Chest CT         → 추적계획: 예정일=2701

   *가 없어도 내용을 보고 섹션을 추론할 것

3. 하위 내용 — 줄 앞에 '-'
   상위 내용과 하위 내용은 같은 맥락으로 분석:
     흉부에서 fibrotic change 보여 Chest CT 시행.     ← 상위
     - Emphysema 및 nodule 확인                       ← 하위 (CT 결과)
     - 흉벽에서 r/o Lipoma                            ← 하위 (CT 결과)

     고지혈증 약 복용 후 부작용은 없었나요?              ← 상위 (질문)
     - 근육통, 혈뇨 등 없었고, 부작용 없었다.           ← 하위 (답변)

   데이터 추출 시:
     - 상위+하위를 묶어서 하나의 맥락으로 이해
     - 영상검사 결과요약에 하위 항목들을 포함시킬 것
     - 문진 내용은 질문+답변을 함께 free_text에 보존
```

### 진단명 파싱 — 핵심 진단명 + 부가 설명 분리
```
의사마다 쓰는 스타일이 다름:
  A 의사: "#고혈압(LSM)"
  B 의사: "#HTN - lifestyle modification"
  C 의사: "#고혈압 / 생활습관교정 중"
  D 의사: "#essential HTN, on LSM"

AI가 분배할 때 분리:
  → 진단명="고혈압" (표준화된 핵심 진단명)
  → 비고="LSM" (부가 설명)
  → 상태="활성"

  "#r/o 고혈압" → 진단명="고혈압", 비고="", 상태="의심"
  "#고혈압 의심" → 진단명="고혈압", 비고="", 상태="의심"

규칙:
  - #으로 시작하면 진단명
  - 괄호, 쉼표, 대시 뒤의 내용 → 비고로 분리
  - r/o, 의심, rule out 등 → 상태를 "의심"으로 설정
  - 핵심 진단명은 통일된 형태로 저장 (통계용)
  - 의사 습관을 바꾸지 않음 — AI가 알아서 정리
```

### 진단 상태 — 3가지
```
활성   → 현재 해당되는 확진 진단 (#고혈압, #고혈압(LSM))
의심   → 아직 확진 안 됨 (#r/o 고혈압)
종결   → 더 이상 해당 안 됨 (오진, 완치 등)

상태 변경 흐름:
  의심 → 활성 (확진됨)
  의심 → 종결 (아닌 것으로 판명)
  활성 → 종결 (완치, 오진 등)
```

### 표준코드(KCD/ICD) 매핑
```
AI가 진단명을 분석할 때 가능하면 KCD/ICD 코드도 매핑:
  "고혈압" → 표준코드="I10"
  "제2형 당뇨" → 표준코드="E11"
  "고지혈증" → 표준코드="E78.5"
  "경동맥 비후" → 표준코드=NULL (정확한 코드 없음)

저장:
  진단명="고혈압", 표준코드="I10"     ← 매핑 가능
  진단명="경동맥 비후", 표준코드=NULL  ← 매핑 불가

규칙:
  - 매핑이 확실한 경우만 코드 입력
  - 확실하지 않으면 빈 문자열 (추측하지 말 것)
  - 의사가 직접 표준코드를 쓴 경우 그대로 사용

의사에게 확인:
  AI: "진단명='고혈압', KCD코드='I10'으로 매핑했습니다. 맞습니까?"
  → 동의/수정 후 저장
```

### 신규 vs 기존 데이터 구분
```
AI가 새 free-text를 받으면:
  1. 기존 검사결과 테이블과 비교
  2. 이미 저장된 수치와 같으면 → "기존참조" → 새로 저장 안 함
  3. 이미 저장된 수치와 다르면 → "변경감지" → 의사에게 확인 요청
  4. 기존에 없는 수치면 → "신규" → 저장

새 데이터     → "이거 넣습니다" (확인만 받고 저장)
충돌 데이터   → "이거 다른데 어떻게 할까요?" (선택 필요)
기존과 동일   → 건너뜀 (자동, 물어보지도 않음)
```

### 날짜 저장 규칙 — 테이블별로 형식이 다름
```
검사결과/영상검사 시행일: YYMMDD 6자리 (실제 시행한 날짜, 정확해야 함)
추적계획 예정일:         YYMM 4자리 허용 ("그 달쯤"이란 의미)
방문기록 방문일:         YYMMDD 6자리 (실제 방문 날짜)

날짜 계산 기준:
  - 모든 날짜 계산은 '이 차트의 방문일'을 기준으로 함 (오늘 날짜가 아님!)
  - 의사가 날짜를 안 쓰면 방문일 사용
  - '약 2개월' → 방문일 기준 2개월 후 (YYMM 형식)
  - 날짜 없는 검사/영상 → 시행일 = 방문일 (YYMMDD)
  - "일주일 후" 같은 표현 → 방문일 기준 계산 → 의사 확인
```

### 추적계획 완료 매칭 — 날짜가 아닌 내용으로
```
추적계획과 검사결과는 날짜 형식이 다름:
  추적계획: 예정일="2604" (4자리, "그 달쯤")
  검사결과: 시행일="260510" (6자리, 실제 날짜)

매칭은 날짜가 아니라 내용(검사항목)으로:
  추적계획: 예정일="2604", 내용="Lab(A1c, Lipid)"
  검사결과: 시행일="260510", 검사항목="A1c"
  → 날짜는 다르지만 내용 일치 → 완료 처리

예: 환자가 4월 예정이었는데 5월에 와서 검사함
  → AI: "추적계획 Lab(A1c) 2604 예정 → 260510에 시행됨 → 완료 처리할까요?"
```

### 과거 검사결과 처리
```
의사가 "260324 LDL 165"라고 쓰면:
  AI가 날짜를 인식 → 검사시행일="260324"로 저장
  (오늘 방문일이 아닌 기록된 날짜로 저장)
```

### 검사결과 저장 원칙 — 항목별 개별 id 부여
```
같은 날 검사해도 항목마다 각각 저장:
  검사id=1, 260310, FBS, 108
  검사id=2, 260310, A1c, 5.8
  검사id=3, 260310, LDL, 162

이유: 연구 시 항목별 추적이 가능해야 하므로
  → "이 환자의 A1c 추이만 보기" = SQL 한 줄로 가능

화면에 보여줄 때는 같은 날짜끼리 묶어서 출력:
  260310  FBS 108, A1c 5.8, LDL 162
  (저장은 따로, 보여주기는 합쳐서)
```

### 외부 기록 사진 분석 (image_reader.py)
```
다른 병원 기록을 사진으로 촬영하여 입력:
  사진 촬영/업로드
      ↓
  AI가 이미지 분석 (OCR + 의미 파악)
      ↓
  JSON으로 정형화:
    {"진단": [...], "검사결과": [...], "처방": [...]}
      ↓
  의사 확인: "이렇게 넣을까요?"
      ↓
  동의 → 기존 DB 함수로 저장 (검사결과추가, 진단추가 등)

구현 원칙:
  - 기존 코드 수정 없음 — 새 모듈(image_reader.py)만 추가
  - from image_reader import 사진분석 → 한 줄로 연결
  - 추출된 데이터는 기존 DB 함수를 그대로 사용하여 저장
  - Claude API의 이미지 분석 기능 활용 (텍스트 분석과 코드 거의 동일)
```

## 2. 수정 시 — 의무기록 보존 원칙 (중요!)

### 핵심 원칙: 원본 보존 + 정정 기록 추가
```
의무기록 법적 원칙:
  - 한 번 작성된 기록은 변경 불가 (법적 증거)
  - 수정이 필요하면 "정정 기록"으로 보완
  - 원본은 그대로 남기고 정정 이력을 덧붙임
  - AI가 임의로 이후 차트를 수정하면 안 됨 (의사가 직접 수정해야 유효)
```

### 유효여부 플래그 (모든 데이터 테이블에 추가)
```
검사결과, 영상검사, 진단, 추적계획, 방문 테이블에:
  유효여부 INTEGER DEFAULT 1  (1=유효, 0=무효)

정정 시:
  기존: 검사id=1, LDL, 173, 유효여부=0  ← 무효 처리 (삭제 안 함)
  정정: 검사id=99, LDL, 163, 유효여부=1  ← 정정값 (유효)

진료/브리핑 시: 유효여부=1만 조회
통계 시:       유효여부=1만 조회 → 오기입 값 제외
이력 보기:     전부 보임 (정정 이력 포함)
```

### 수정 흐름 — 의사가 직접 수정
```
의사: "260301 LDL 173이 오기입이다. 163이 맞다."
  ↓
시스템:
  1. 기존 검사id=1 → 유효여부=0으로 변경
  2. 새 검사결과 추가: LDL 163, 유효여부=1
  3. 방문 free_text에 정정 기록 추가:
     "[정정] LDL 173 → 163 (사유: 오기입)"
  ↓
시스템이 영향받는 이후 차트를 찾아서 알려줌:
  "다음 차트에서 LDL 173을 참조하고 있습니다:
    - 260415 차트: '이전 LDL 173 참고하여...'
    - 260601 차트: '스타틴 복용 후 LDL 94로 감소 (173→94)'
   확인하시겠습니까?"
  ↓
의사가 각 차트를 열어서 직접 수정 (AI가 자동 수정하면 안 됨!)
  → 각 수정도 정정 기록으로 남음

AI의 역할:
  ✅ 영향받는 차트를 찾아줌
  ✅ 정정 기록을 자동 작성
  ❌ 이후 차트를 임의로 수정 (의무기록 위반!)
```

### free-text 수정
```
원본 free_text는 보존 (절대 덮어쓰지 않음)
수정 내용은 "정정 기록"으로 붙임:
  "[260601 정정] 260301 기록의 LDL 173 → 163 정정"
```

## 3. 삭제 시 — 무효 처리 원칙 (삭제가 아님!)

### 핵심 원칙: 실제 삭제 안 함 + 무효 플래그 처리
```
의무기록은 삭제하면 안 됨 → "무효" 처리로 대체
원본은 남기되 유효여부=0으로 표시
통계/브리핑에서는 제외, 이력에서는 보임
```

### 방문 기록 무효화
```
의사: "260301 기록을 삭제하겠습니다"
시스템:
  "이 날짜의 기록을 무효 처리합니다:
   진료 기록, 진단 2건, 검사 3건, 영상 1건
   무효 처리하시겠습니까? 사유를 입력하세요."
→ 사유 입력 → 승인

백그라운드:
  방문 유효여부=0 + 정정 기록 추가
  해당 방문의 진단/검사/영상/추적도 유효여부=0
  이후 차트는 건드리지 않음

화면 표시:
  260301 차트 (무효 처리됨):
  ──────────────────
  (취소선) LDL 173, 고지혈증 약 시작
  [무효] 260601 처리. 사유: 오기입
  ──────────────────
```

### 진단 무효화
```
진단 삭제 대신 무효 처리:
  기존 진단 유효여부=0 → 브리핑/통계에서 제외
  권장: 무효 처리보다 상태 변경 (활성→종결) 우선 고려
```

### 검사결과/영상검사 무효화
```
삭제 대신 유효여부=0으로 변경
AI가 이후 차트에서 참조하는 곳을 찾아서 알려줌:
  "이 검사를 참조하는 차트 2건이 있습니다. 확인하시겠습니까?"
→ 의사가 직접 이후 차트 수정 (AI가 자동 수정 안 함)
```

### 환자 전체 삭제
```
환자 전체 삭제는 예외적으로 실제 삭제 허용 (테스트 데이터 등)
→ "이 환자의 모든 기록이 영구 삭제됩니다. 'yes' 입력:"
→ 6개 테이블에서 해당 환자 기록 전부 삭제
```

### 수정/삭제 공통 흐름 (핵심!)
```
수정이든 삭제든 동일한 패턴:

  1. 해당 기록 처리 (수정=원본 무효+정정본 추가 / 삭제=무효 플래그)
  2. 원본 + 사유 보관
  3. AI가 영향받는 이후 차트 검색 → 의사에게 제시
     "다음 차트에서 관련 내용을 참조하고 있습니다:
      - 260415 차트: '이전 LDL 173 참고하여...'
      확인/수정이 필요할 수 있습니다."
  4. 의사가 해당 차트를 열어서 직접 수정 → 같은 패턴 반복
  5. 더 이상 영향받는 차트 없으면 끝

AI의 역할:
  ✅ 영향받는 차트를 찾아줌 (영향받는_차트_검색 함수)
  ✅ 정정/무효 기록을 자동 작성
  ❌ 이후 차트를 임의로 수정 (의무기록 위반!)
  → 의사가 직접 수정해야만 유효한 의무기록
```

## 4. 조회/브리핑 시 — 환자 이력 분석

### AI 브리핑 형식
```
의사가 환자를 선택하면:
  1. SQL로 해당 환자의 전체 데이터 조회 (JOIN)
  2. AI에게 전달
  3. AI가 아래 형식으로 브리핑 생성:

[진단/문제목록]
- 활성 진단과 의심 진단을 나열

[검사결과 추이]
- 항목별 수치 변화 (화살표로 추이 표시)

[임상 요약]
한 문단으로 자연스럽게 서술:
  "OOO로 OOO하던 환자로, 이전 OOO에서 OOO했고,
   금일 OOO 위해 내원하였음. 따라서 이번 방문 시에는
   OOO를 확인하고 OOO해야 함."

[이번 방문 확인사항]
진단/문제별로 묶어서 작성 (문제 중심 접근):

고혈압
- amlodipine 5mg qd 복용 중 → 혈압 추이 확인 및 지속 상승 시 약제 조절 고려

고지혈증
- 이전 경동맥 초음파에서 IMT max 1.1mm로 LDL<100 조절 목표
- 이전 Lab 상 TG 180으로 상승 소견보여 금번 Lab 확인 후 약제 추가 조절 고려 중

당뇨 전단계
- FBS 상승 및 HbA1c 5.8으로 식습관 조절하면서 경과 관찰 중

금일 Lab(A1c, Lipid) 결과 확인 및 혈압 확인하여 약제 조절 하여야 함.

작성 원칙:
  - 각 진단별로 '왜 → 무엇을 → 어떻게'의 맥락이 보여야 함
  - 이전 소견(검사결과, 영상결과)을 근거로 이번 조치를 연결
  - 마지막에 금일 해야 할 핵심 업무를 한 문장으로 요약
  - 의학 약어 사용 가능 (qd, f/u, Tx, Dx, LSM 등)
  - 불필요한 반복 금지

시제 규칙 — 매우 중요:
  브리핑은 '오늘 환자를 보기 전'에 읽는 문서.
  이전 기록과 오늘 할 일을 명확히 구분할 것.

  잘못된 예:
    갑상선기능저하증
    * TSH 1.8로 정상범위 내 잘 조절됨 ← 이미 확인한 것처럼
    * 4개월 후 TSH, FT4 추적 예정 ← 이미 결정한 것처럼

  올바른 예:
    갑상선기능저하증
    * 레보티록신 75mcg qd 복용 중으로 이전 TSH 1.8로 잘 조절되고 있음 ← 이전 상태
    * 금일 혈액검사 시행하여 TSH, fT4 추적 예정 ← 오늘 할 일
    TSH 잘 유지되던 환자로 오늘 검사 시행하여 용량 조절 필요할지 확인해야 함.

  원칙:
    - 이전 검사 결과/소견 → '이전', '지난 방문', '기존' 등으로 과거 표현
    - 오늘 할 일 → '금일', '오늘', '이번 방문' 등으로 미래/현재 표현
    - 마지막 요약 → '오늘 OOO해서 OOO 확인해야 함'으로 행동 지시형
```

### 차트 검토 (추출과 통합)
```
별도 검토 단계 없이 차트 분석 시 동시 수행:
  AI가 데이터 추출 + 검토 의견을 한 번에 제공

검토의견 유형:
  - "누락": 해야 하는데 빠진 항목 → 추적계획에 추가
  - "추적": 기존 추적계획 중 완료 처리 권장 → 추적계획 완료 처리
  - "주의": 약물/수치 관련 주의사항 → 방문 메모에 기록
  - "추천": 추가로 고려할 만한 조치 → 추적계획에 추가
```

### 누락 체크
```
AI가 자동으로 확인:
  - "260112에 검사했는데 이후 방문에서 결과 확인 언급이 없습니다"
  - "추적계획 Lab(Lipid, LFT) 2605 예정인데 미완료입니다"
  - "마지막 BMD 검사 후 1년 경과했습니다"
```

### AI 보완 차트 작성 (Week 5~6에서 구현 예정)
```
데이터 추출 + 검토의견 동의 후, AI가 의사 스타일로 보완된 차트를 작성 (2번째 API 호출)

전체 흐름:
  1. 의사가 free-text 작성
  2. AI가 데이터 추출 + 검토의견 → 의사 동의/반려 (현재 구현됨)
  3. AI가 동의된 내용을 원본 차트에 녹여서 보완본 작성
  4. 의사에게 원본 vs 보완본 나란히 보여줌
     → 변경 부분 [+] [수정] 표시
  5. 의사 선택:
     1) 원본 유지
     2) 보완본 승인
     3) 보완본 직접 편집 후 저장
  6. 저장 시 [+] [수정] 표시 자동 제거 → 깨끗한 최종본만 저장

보완 차트 AI 프롬프트 원칙:
  - 의사의 기존 문체/스타일 유지 (*, **, - 등 그대로)
  - 오타, 수치 오류, 부정확한 표현은 수정
  - 동의된 추천사항은 기존 차트 흐름에 자연스럽게 삽입
  - 의사가 의도적으로 작성한 내용(환자 교육, 설명 등)은 임의로 삭제하지 말 것
  - 변경된 부분은 [+] 또는 [수정] 표시로 어디가 바뀌었는지 알 수 있게
```

## 6. 연구 시 — 데이터 추출 및 분석

### 정형 데이터 검색 (SQL)
```
"60세 이상 고혈압 환자 중 LDL 160 이상인 사람"
  → AI가 SQL로 변환:
    SELECT ... FROM 환자 JOIN 검사결과 ...
    WHERE 나이 >= 60 AND 진단 = '고혈압' AND LDL >= 160
  → 바로 결과 반환
```

### 비정형 데이터 분석 (AI) — free_text 분석
```
모드 1, 2에 통합 (별도 메뉴 아님).
AI가 연구 질문을 보고 free_text 분석이 필요한지 자동 판단.

free_text 분석이 필요한 경우:
  "부작용 호소한 환자" → 증상 표현이 다양 (근육통, myalgia, 다리 뻣뻣...)
  "금연 성공 패턴" → 경과 기록 분석 필요
  "환자 교육 내용 정리" → free_text에만 있는 정보

free_text 분석 불필요:
  "LDL 평균" → 검사결과 테이블만으로 충분
  "고혈압 환자 수" → 진단 테이블만으로 충분

범위 선택:
  1. SQL 선추림 후 분석 (권장, 비용 절감):
     SQL로 후보 환자 추림 → 추려진 환자의 free_text만 AI 분석
     예: 1000명 중 스타틴 처방 50명만 추림 → 비용 1/20
  
  2. 전체 환자 분석 (정밀, 비용 높음):
     모든 환자의 free_text를 AI가 분석
     논문 발표용 등 정밀도가 필요할 때

비식별화 적용:
  AI에게 보낼 때: 이름→환자001, 생년월일→나이대
  결과 출력 시: 비식별→실명 복원
  비용 안내 후 진행 여부 확인

유효여부=1인 방문 기록만 대상
```

### 통계/분석 — 두 가지 모드
```
[모드 1: AI 자동 분석]
의사가 자연어로 질문:
  "스타틴 복용군과 비복용군의 LDL 변화 비교해줘"
    ↓
  AI가 알아서:
    1. 데이터 추출 (SQL)
    2. 비식별화 (이름→ID)
    3. 적절한 통계 기법 선택 + 근거 설명
    4. 분석 실행
    5. 결과 + 그래프 제시
    6. "paired t-test를 사용했습니다. 이유: 동일 환자의 전후 비교"

[모드 2: 의사가 단계별 선택]
  Step 1: 데이터 선택 (환자군, 변수, 기간)
  Step 2: 그룹 설정 (그룹 나누기, 무작위 배정 여부)
  Step 3: 통계 기법 선택
    - t-test, paired t-test (두 그룹 평균 비교)
    - chi-square (범주형 비교)
    - ANOVA (세 그룹 이상 비교)
    - 회귀분석, 로지스틱 회귀
    - 생존분석 (Kaplan-Meier)
    - "AI 추천 받기" 옵션
  Step 4: 결과 (p-value, 신뢰구간, 그래프)
  Step 5: 내보내기 (CSV, Excel, 그래프 이미지)

연구 도구:
  비식별화:   이름→ID, 생년월일→나이대 변환
  무작위 배정: 랜덤 그룹 배정 (treatment vs control)
  라벨링:     조건별 자동 분류 (예: LDL≥160 → "고위험군")
  내보내기:   CSV, Excel, SPSS 형식, 그래프 이미지(PNG/PDF)
  시각화:     box plot, violin plot, scatter plot, line chart

라이브러리: pandas, scipy.stats, statsmodels, pingouin, matplotlib, seaborn
```

### 표준코드 기반 연구 통계 (다중 사용자 환경)
```
연구 시 진단 통계를 돌릴 때 2단계로 처리:

1단계: 표준코드가 있는 진단 → SQL로 자동 포함 (확실)
  SELECT COUNT(*) FROM 진단 WHERE 표준코드 = 'I10'
  → 고혈압 20명

2단계: 표준코드가 NULL인 진단 → AI가 후보 추려서 의사에게 제시
  SELECT 진단명 FROM 진단 WHERE 표준코드 IS NULL
  → AI: "표준코드가 없는 진단 중 고혈압 관련 가능성:
    □ 혈압 경계치 (3명) — 포함할까요?
    □ 경동맥 비후 (2명) — 포함할까요?
    □ 검진 이상 (5명) — 관련 없어 보임"
  → 의사 판단: "혈압 경계치는 포함, 나머지는 제외"
  → 최종: 고혈압 23명

NULL은 빠지는 게 아니라 "따로 모아서 확인받는 것"
  코드 있음  = 확진 → 자동 포함
  코드 NULL = 리뷰 대상 → AI가 후보 추림 → 의사 판단 → 포함/제외
```

free_text 분석:   temperature 0.3 (다양한 증상 해석 허용)
결과 해석:        temperature 0.3 (자연스러운 설명)

```
SQL의 역할:
  - 정형 데이터 저장/검색/정렬/통계
  - 빠른 조건 검색 (실시간)
  - 데이터 무결성 보장 (FOREIGN KEY 등)

AI의 역할:
  - free-text 분석 → 정형화 → 테이블 분배
  - 차트 스타일 인식 (*, **, - 등)
  - 진단명 파싱 (핵심+비고 분리, 표준코드 매핑)
  - 기존 데이터와 비교 → 중복/오류 감지
  - 비정형 데이터(free-text)에서 정보 추출
  - 브리핑 생성 (문제 중심, 임상 요약 스타일)
  - 차트 검토 (누락/주의/추천 — 추출과 동시 수행)
  - 날짜 자동 변환 (방문일 기준, 테이블별 형식 구분), 자연어→SQL 변환
  - 추적계획 완료 매칭 (날짜가 아닌 내용으로)
  - 외부 기록 이미지 분석 (OCR)

의사의 역할 (AI가 대체 불가):
  - free-text 작성 (원본 진료 기록)
  - 의학적 판단 (진단, 처방, 검사 결정)
  - AI 제안의 동의/반려 (최종 결정권)
  - 테이블 설계 (의학적 데이터 구조)
  - 프롬프트 설계 (AI에게 어떤 역할을 시킬지)
```
