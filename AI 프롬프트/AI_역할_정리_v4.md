# AI 역할 정리 v4 (프로젝트 설계 기준)

## 1. 데이터 입력 시 — Free-text 분석 및 분배

### 기본 흐름
```
의사가 free-text로 차트 작성 (기존 습관 그대로)
    ↓
AI가 한 번의 API 호출로 두 가지를 동시 수행:
  ① 데이터 추출: free-text에서 정형 데이터 분류
  ② 차트 검토: 누락/오류/추가 제안
    ↓
각 테이블로 분배:
  - 진단 → 진단 테이블
  - 검사 수치 → 검사결과 테이블
  - 영상/특수검사 → 영상검사 테이블
  - 추적 계획 → 추적계획 테이블
  - 활력징후/생활습관 → 방문 테이블
  - 원본 free-text → 방문 테이블에 그대로 보존
    ↓
의사에게 추출 데이터 + 검토 의견을 한꺼번에 보여주기
    ↓
동의/반려 → 저장 (API 1번, 확인 1번, 저장 1번)
```

### AI 분석 실패 시 대응
```
AI 분석 실패 (API 오류, 서버 장애 등):
  → free_text만 방문 테이블에 저장 (분석완료=0)
  → 안내: "입력하신 차트는 free-text로 먼저 저장되었습니다.
           다음 접속 시 AI가 자동으로 분석을 시도합니다."

다음 로그인 시:
  → 프로그램 시작 시 분석완료=0인 차트 자동 체크
  → "미분석 차트 N건이 있습니다. 지금 분석할까요? (y/n)"
  → y: AI 분석 → 확인 → 저장 → 분석완료=1
  → n: 다음 접속 시 다시 물어봄
```

### 차트 스타일 인식 규칙
```
의사의 차트 작성 스타일:

1. 섹션 구분 — * (별표 1개)
   [섹션1: 진단/문제목록/검사결과]
   #진단명, 검사 수치 등
   *
   [섹션2: 신체검진/생활습관]
   BP, 키, 몸무게, 흡연, 음주, 운동 등
   *
   [섹션3: 진료내용]
   진료 경과, 설명, 문진 등 (free-text)
   *
   [섹션4: 계획]
   **약 처방, **추적검사 등

2. 별표 규칙:
   * (1개)  = 섹션 구분자
   ** (2개) = 계획 항목 (약 처방, 추적검사 모두 동일)
     예: **약 2개월
     예: **2604 Lab(A1c, Lipid)
     예: **2701 Chest CT
   *가 없어도 내용을 보고 섹션을 추론할 것

3. 하위 내용 — 줄 앞에 '-'
   상위 내용과 하위 내용은 같은 맥락으로 분석:
     흉부에서 fibrotic change 보여 Chest CT 시행.     ← 상위
     - Emphysema 및 nodule 확인                       ← 하위 (CT 결과)
     - 흉벽에서 r/o Lipoma                            ← 하위 (CT 결과)

     고지혈증 약 복용 후 부작용은 없었나요?              ← 상위 (질문)
     - 근육통, 혈뇨 등 없었고, 부작용 없었다.           ← 하위 (답변)

   데이터 추출 시:
     - 상위+하위를 묶어서 하나의 맥락으로 이해
     - 영상검사 결과요약에 하위 항목들을 포함시킬 것
     - 문진 내용은 질문+답변을 함께 free_text에 보존
```

### 진단명 파싱 — 핵심 진단명 + 부가 설명 분리
```
의사마다 쓰는 스타일이 다름:
  A 의사: "#고혈압(LSM)"
  B 의사: "#HTN - lifestyle modification"
  C 의사: "#고혈압 / 생활습관교정 중"
  D 의사: "#essential HTN, on LSM"

AI가 분배할 때 분리:
  → 진단명="고혈압" (표준화된 핵심 진단명)
  → 비고="LSM" (부가 설명)
  → 상태="활성"

  "#r/o 고혈압" → 진단명="고혈압", 비고="", 상태="의심"
  "#고혈압 의심" → 진단명="고혈압", 비고="", 상태="의심"

규칙:
  - #으로 시작하면 진단명
  - 괄호, 쉼표, 대시 뒤의 내용 → 비고로 분리
  - r/o, 의심, rule out 등 → 상태를 "의심"으로 설정
  - 핵심 진단명은 통일된 형태로 저장 (통계용)
  - 의사 습관을 바꾸지 않음 — AI가 알아서 정리
```

### 진단 상태 — 3가지
```
활성   → 현재 해당되는 확진 진단 (#고혈압, #고혈압(LSM))
의심   → 아직 확진 안 됨 (#r/o 고혈압)
종결   → 더 이상 해당 안 됨 (오진, 완치 등)

상태 변경 흐름:
  의심 → 활성 (확진됨)
  의심 → 종결 (아닌 것으로 판명)
  활성 → 종결 (완치, 오진 등)
```

### 표준코드(KCD/ICD) 매핑
```
AI가 진단명을 분석할 때 가능하면 KCD/ICD 코드도 매핑:
  "고혈압" → 표준코드="I10"
  "제2형 당뇨" → 표준코드="E11"
  "고지혈증" → 표준코드="E78.5"
  "경동맥 비후" → 표준코드=NULL (정확한 코드 없음)

저장:
  진단명="고혈압", 표준코드="I10"     ← 매핑 가능
  진단명="경동맥 비후", 표준코드=NULL  ← 매핑 불가

규칙:
  - 매핑이 확실한 경우만 코드 입력
  - 확실하지 않으면 빈 문자열 (추측하지 말 것)
  - 의사가 직접 표준코드를 쓴 경우 그대로 사용

의사에게 확인:
  AI: "진단명='고혈압', KCD코드='I10'으로 매핑했습니다. 맞습니까?"
  → 동의/수정 후 저장
```

### 신규 vs 기존 데이터 구분
```
AI가 새 free-text를 받으면:
  1. 기존 검사결과 테이블과 비교
  2. 이미 저장된 수치와 같으면 → "기존참조" → 새로 저장 안 함
  3. 이미 저장된 수치와 다르면 → "변경감지" → 의사에게 확인 요청
  4. 기존에 없는 수치면 → "신규" → 저장

새 데이터     → "이거 넣습니다" (확인만 받고 저장)
충돌 데이터   → "이거 다른데 어떻게 할까요?" (선택 필요)
기존과 동일   → 건너뜀 (자동, 물어보지도 않음)
```

### 날짜 변환 규칙
```
- YYMMDD 6자리가 기본
- 의사가 YYMM 4자리만 쓰면 DD는 01로 채움 (예: 2604 → 260401)
- 의사가 날짜를 안 쓰면 오늘 날짜 사용
- "일주일 후" 같은 표현 → 오늘 날짜 기준 계산 → 의사 확인
- "약 2개월" → 오늘 날짜 기준으로 2개월 후 계산
```

### 과거 검사결과 처리
```
의사가 "260324 LDL 165"라고 쓰면:
  AI가 날짜를 인식 → 검사시행일="260324"로 저장
  (오늘 방문일이 아닌 기록된 날짜로 저장)
```

### 검사결과 저장 원칙 — 항목별 개별 id 부여
```
같은 날 검사해도 항목마다 각각 저장:
  검사id=1, 260310, FBS, 108
  검사id=2, 260310, A1c, 5.8
  검사id=3, 260310, LDL, 162

이유: 연구 시 항목별 추적이 가능해야 하므로
  → "이 환자의 A1c 추이만 보기" = SQL 한 줄로 가능

화면에 보여줄 때는 같은 날짜끼리 묶어서 출력:
  260310  FBS 108, A1c 5.8, LDL 162
  (저장은 따로, 보여주기는 합쳐서)
```

### 외부 기록 사진 분석 (image_reader.py)
```
다른 병원 기록을 사진으로 촬영하여 입력:
  사진 촬영/업로드
      ↓
  AI가 이미지 분석 (OCR + 의미 파악)
      ↓
  JSON으로 정형화:
    {"진단": [...], "검사결과": [...], "처방": [...]}
      ↓
  의사 확인: "이렇게 넣을까요?"
      ↓
  동의 → 기존 DB 함수로 저장 (검사결과추가, 진단추가 등)

구현 원칙:
  - 기존 코드 수정 없음 — 새 모듈(image_reader.py)만 추가
  - from image_reader import 사진분석 → 한 줄로 연결
  - 추출된 데이터는 기존 DB 함수를 그대로 사용하여 저장
  - Claude API의 이미지 분석 기능 활용 (텍스트 분석과 코드 거의 동일)
```

## 2. 수정 시 — 데이터 유형별 다른 처리

### 검사결과/영상검사 (객관적 사실 — 원본 1개)

**중요: 같은 날 검사도 항목별로 각각 id가 부여됨**
```
DB 저장 (항목별 개별 id):
  검사id=1, 260310, FBS, 108
  검사id=2, 260310, A1c, 5.8
  검사id=3, 260310, LDL, 162

화면 출력 (같은 날짜끼리 묶어서):
  260310  FBS 108, A1c 5.8, LDL 162
```

**수정 시 — AI가 항목별로 비교하여 바뀐 것만 확인:**
```
의사가 free-text 수정: "260310 FBS 110, A1c 5.8, LDL 162"
                            ↑ 108→110 바뀜

AI가 항목별 비교:
  검사id=1, FBS: 기존 108 → 새 110 (변경 감지!)
  검사id=2, A1c: 기존 5.8 → 새 5.8 (동일 → 건너뜀)
  검사id=3, LDL: 기존 162 → 새 162 (동일 → 건너뜀)

AI: "FBS가 108→110으로 바뀌었는데 검사결과도 수정할까요?"
  → 승인 시 검사id=1만 수정 → 전체 자동 반영
  → 바뀌지 않은 항목은 건드리지 않음
```

### 진단 (시간에 따라 변함 — 이력 보존 필요)
```
진단 수정 시:
  → AI가 같은 진단명을 가진 모든 기록을 목록으로 보여줌:
    □ 진단id=1, 방문id=1 (260110 초진)
    □ 진단id=2, 방문id=2 (260210 경과)
    □ 진단id=3, 방문id=3 (260310 경과)
    ☑ 진단id=4, 방문id=4 (260415 약 시작)

  → 선택지 3가지:
    1. 이 차트만 수정 (1개)
    2. 선택한 차트들 수정 (직접 선택)
    3. 같은 진단 전부 수정

  → 예시: "고혈압"을 약 시작 전 차트만 "고혈압 의심"으로 바꾸고 싶을 때
    → 2번 선택 → 진단id 1,2,3 체크 → "고혈압 의심"으로 수정
    → 진단id=4(약 시작)는 "고혈압" 유지

진단 변경의 권장 방식:
  기존 진단 상태="종결" + 새 진단 추가 (이력 보존)
  예: "고혈압 의심"→종결 + "고혈압 확진" 새로 추가
```

### free-text 수정 시 전체 흐름
```
의사가 free-text 수정
  → AI가 수정 전후를 비교
  → 변경된 항목 감지:
    - 검사 수치 변경 → 항목별로 비교, 바뀐 것만 "수정할까요?" 확인
      (동일한 항목은 건너뜀, 변경된 항목만 확인)
    - 진단 변경 → "이 차트만? 선택한 차트들? 전부?" (3가지 선택지)
    - 추적계획 변경 → "추적계획 테이블도 수정할까요?"
    - 영상검사 변경 → 항목별로 비교, 바뀐 것만 확인
  → 각각 의사 확인 후 반영
  → 안 바뀐 항목은 자동 건너뜀 (물어보지 않음)
```

## 3. 삭제 시 — 데이터 보호 원칙

### 방문 기록 삭제
```
방문 삭제 시:
  → free_text만 삭제
  → 검사결과/영상검사: 절대 삭제 안 함 (방문과 독립)
  → 진단: 유지 (환자의 진단은 차트 하나와 무관)
  → 추적계획: 유지 (계획은 아직 유효할 수 있음)
```

### 진단 삭제
```
진단 삭제 시:
  → AI가 같은 진단명을 가진 모든 기록을 목록으로 보여줌
  → 선택지 3가지:
    1. 이 차트의 진단만 삭제 (1개)
    2. 선택한 차트들의 진단 삭제 (직접 선택)
    3. 같은 진단명 전부 삭제
  → 권장: 삭제보다 상태 변경 (활성→종결)
```

### 검사결과/영상검사 삭제
```
삭제 요청 시 AI가 같은 날짜의 항목 목록을 보여줌:

AI: "260310에 3건의 검사결과가 있습니다:
  □ 검사id=1, FBS 108
  □ 검사id=2, A1c 5.8
  □ 검사id=3, LDL 162

  1. 전부 삭제
  2. 선택해서 삭제"

→ 전부 선택 시: 3건 다 삭제
→ 선택 시: 원하는 항목만 체크해서 삭제
→ AI: "삭제하면 다른 차트에서도 참조할 수 없습니다. 진행할까요?"
→ 승인 시 삭제

영상검사도 동일한 방식:
AI: "260415에 1건의 영상검사가 있습니다:
  □ 영상id=1, 경초: Rt. IMT 1.1mm...
  삭제할까요?"
→ 승인 시 삭제
```

### 환자 전체 삭제
```
→ AI: "이 환자의 모든 기록(방문, 진단, 검사, 영상, 추적)이 삭제됩니다. 진행할까요?"
→ 승인 시 6개 테이블에서 해당 환자 기록 전부 삭제
```

## 4. 조회/브리핑 시 — 환자 이력 분석

### AI 브리핑 형식
```
의사가 환자를 선택하면:
  1. SQL로 해당 환자의 전체 데이터 조회 (JOIN)
  2. AI에게 전달
  3. AI가 아래 형식으로 브리핑 생성:

[진단/문제목록]
- 활성 진단과 의심 진단을 나열

[검사결과 추이]
- 항목별 수치 변화 (화살표로 추이 표시)

[임상 요약]
한 문단으로 자연스럽게 서술:
  "OOO로 OOO하던 환자로, 이전 OOO에서 OOO했고,
   금일 OOO 위해 내원하였음. 따라서 이번 방문 시에는
   OOO를 확인하고 OOO해야 함."

[이번 방문 확인사항]
진단/문제별로 묶어서 작성 (문제 중심 접근):

고혈압
* amlodipine 5mg qd 복용 중 → 혈압 추이 확인 및 지속 상승 시 약제 조절 고려

고지혈증
* 이전 경동맥 초음파에서 IMT max 1.1mm로 LDL<100 조절 목표
* 이전 Lab 상 TG 180으로 상승 소견보여 금번 Lab 확인 후 약제 추가 조절 고려 중

당뇨 전단계
```

## 6. 연구 시 — 데이터 추출 및 분석

### 정형 데이터 검색 (SQL)
```
"60세 이상 고혈압 환자 중 LDL 160 이상인 사람"
  → AI가 SQL로 변환:
    SELECT ... FROM 환자 JOIN 검사결과 ...
    WHERE 나이 >= 60 AND 진단 = '고혈압' AND LDL >= 160
  → 바로 결과 반환
```

### 비정형 데이터 분석 (AI)
```
"두통을 호소한 환자를 찾아줘"
  → SQL로 free_text가 있는 방문 기록 조회
  → AI가 free_text를 읽고 "두통" 관련 기록 추출

"이 환자의 검사 목적을 정리해줘"
  → 검사결과에서 검사시행일 확인
  → 해당 날짜 전후 방문 free_text를 AI가 읽고 목적 추출
```

### 통계/분석
```
"A1c 추이를 분석해줘"
  → SQL: SELECT 검사시행일, 결과값 FROM 검사결과 WHERE 검사항목='A1c'
  → AI: "6.4(2508) → 6.1(2601) → 5.8(2604), 개선 추세"

"진단별 환자 수 통계"
  → SQL: SELECT 진단명, COUNT(*) FROM 진단 GROUP BY 진단명
```

### 표준코드 기반 연구 통계 (다중 사용자 환경)
```
연구 시 진단 통계를 돌릴 때 2단계로 처리:

1단계: 표준코드가 있는 진단 → SQL로 자동 포함 (확실)
  SELECT COUNT(*) FROM 진단 WHERE 표준코드 = 'I10'
  → 고혈압 20명

2단계: 표준코드가 NULL인 진단 → AI가 후보 추려서 의사에게 제시
  SELECT 진단명 FROM 진단 WHERE 표준코드 IS NULL
  → AI: "표준코드가 없는 진단 중 고혈압 관련 가능성:
    □ 혈압 경계치 (3명) — 포함할까요?
    □ 경동맥 비후 (2명) — 포함할까요?
    □ 검진 이상 (5명) — 관련 없어 보임"
  → 의사 판단: "혈압 경계치는 포함, 나머지는 제외"
  → 최종: 고혈압 23명

NULL은 빠지는 게 아니라 "따로 모아서 확인받는 것"
  코드 있음  = 확진 → 자동 포함
  코드 NULL = 리뷰 대상 → AI가 후보 추림 → 의사 판단 → 포함/제외
```

## 7. AI 안전 원칙

### 할루시네이션 대응
```
- AI에게 "확정"이 아니라 "제안 + 근거"를 요청
- 95% 정확해도 5%가 위험할 수 있음
- 신뢰가 쌓인 후의 오류가 가장 잡기 어려움
```

### Temperature 설정
```
차트 분석/검토:   temperature 0.1 (일관된 판단)
브리핑 생성:      temperature 0.1 (일관된 브리핑)
연구 데이터:      temperature 0.3~0.5 (다양한 시각 허용)
```

## 역할 분담 요약

```
SQL의 역할:
  - 정형 데이터 저장/검색/정렬/통계
  - 빠른 조건 검색 (실시간)
  - 데이터 무결성 보장 (FOREIGN KEY 등)

AI의 역할:
  - free-text 분석 → 정형화 → 테이블 분배
  - 차트 스타일 인식 (*, **, - 등)
  - 진단명 파싱 (핵심+비고 분리, 표준코드 매핑)
  - 기존 데이터와 비교 → 중복/오류 감지
  - 비정형 데이터(free-text)에서 정보 추출
  - 브리핑 생성 (문제 중심, 임상 요약 스타일)
  - 차트 검토 (누락/주의/추천 — 추출과 동시 수행)
  - 날짜 자동 변환, 자연어→SQL 변환
  - 외부 기록 이미지 분석 (OCR)

의사의 역할 (AI가 대체 불가):
  - free-text 작성 (원본 진료 기록)
  - 의학적 판단 (진단, 처방, 검사 결정)
  - AI 제안의 동의/반려 (최종 결정권)
  - 테이블 설계 (의학적 데이터 구조)
  - 프롬프트 설계 (AI에게 어떤 역할을 시킬지)
```
