## 2단계: 데이터베이스 (Day 11~25)

> JSON 파일의 한계를 느끼고, 본격적인 데이터베이스를 배우는 단계입니다.
> 환자 데이터를 체계적으로 저장하고 검색하는 법을 익힙니다.

### Day 11: 왜 데이터베이스가 필요한가
- JSON 파일의 한계 (검색 느림, 동시 접근 불가, 구조 강제 불가)
- 데이터베이스의 개념 (엑셀처럼 표로 저장, 하지만 훨씬 강력)
- SQLite 소개 (설치 불필요, Python에 내장)
- 실습: 첫 데이터베이스 만들기, 테이블 생성

### Day 12: SQL 기초 — 데이터 넣고 꺼내기
- INSERT: 데이터 추가 (환자 등록)
- SELECT: 데이터 조회 (환자 검색)
- WHERE: 조건 검색 ("고혈압 환자만 보여줘")
- ORDER BY: 정렬 ("나이순으로 보여줘")
- 실습: 환자 10명 등록하고 다양한 조건으로 검색

### Day 13: SQL 중급 — 데이터 수정과 삭제
- UPDATE: 데이터 수정 (진단명 변경, 혈압 갱신)
- DELETE: 데이터 삭제
- COUNT, AVG, MAX, MIN: 통계 함수
- GROUP BY: 그룹별 통계 ("진단별 환자 수")
- 실습: 환자 데이터 수정, 진단별 환자 수 통계

### Day 14: 테이블 설계 — 의료 데이터 구조 잡기
- 테이블 간 관계 이해 (환자 ↔ 진료기록 ↔ 검사결과)
- Primary Key, Foreign Key의 개념
- 1:N 관계 (환자 1명 → 진료기록 여러 개)
- 실습: 환자 테이블 + 진료기록 테이블 설계

### Day 15: 테이블 설계 실습
- 선생님의 실제 차트 구조를 기반으로 테이블 설계

- 진료기록(visits) 테이블

### Day 16: JOIN — 테이블 연결하기
- JOIN의 개념 ("환자 이름과 그 환자의 진료기록을 함께 보여줘")
- INNER JOIN, LEFT JOIN
- 여러 테이블을 연결한 복합 조회
- 실습: "김철수의 모든 진료기록과 검사결과를 함께 조회"
