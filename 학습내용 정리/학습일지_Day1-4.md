## Day 3 ✅

### 배운 개념

**input()**
- 사용자에게 질문하고 답을 기다리는 함수
- `이름 = input("환자 이름: ")`

**int()**
- 입력받은 글자를 숫자로 변환
- `나이 = int(input("나이: "))`

**while True + break**
- "끝이라고 할 때까지 계속 반복"
- break로 반복문 탈출

**함수 (def, return)**
- 자주 쓰는 작업에 이름을 붙여 저장 (처방 세트)
- `def 혈압판정(수축기, 이완기):`
- return: 함수가 결과를 돌려줌

**f-string**
- 문자열 안에 변수를 직접 끼워넣기
- `f"{i}번 환자: {이름}"`

**enumerate()**
- 리스트에서 꺼내면서 순번도 함께 부여
- `for i, 환자 in enumerate(환자목록, 1):`

**.append()**
- 리스트에 새 항목 추가
- `환자목록.append(환자)`

**len()**
- 리스트에 몇 개가 들어있는지 세기
- `len(환자목록)`

### 핵심 질문과 이해

**들여쓰기의 의미**
- Q: "들여쓰기는 하나의 항목으로 동작한다는 건가?"
- A: 맞음. "이 코드는 위에 있는 것의 소속이다"라는 뜻. 콜론(`:`)으로 끝나는 줄 다음에 들여쓰기가 와야 함.

**실행 순서**
- Q: "while 명령 아래 작업이 완료되어야 그 아래 출력이 실행되나? 항상 위에서 아래 순서로?"
- A: 기본은 위→아래. 예외: 반복문(위로 돌아감), 함수(정의 시 저장만, 호출 시 실행), 조건문(맞는 줄만 실행)

**데이터 흐름 (2단계 전달)**
- Q: "함수가 딕셔너리의 변수를 가져오고, 딕셔너리는 input의 값을 가져오는 2단계 구조인가?"
- A: 맞음. input → 변수 → 딕셔너리에 포장 → 함수가 딕셔너리에서 꺼냄. 결과는 같지만 내부적으로 2단계를 거침.

**for A in B의 정확한 의미**
- Q: "A를 B에서 꺼내라가 아닌가?"
- A: "B에서 꺼내서 A라고 이름을 붙여라". A는 아무 이름이나 가능. 리스트에 원래 A라는 이름이 있어서가 아니라, for문이 돌 때마다 새로 이름을 붙여주는 것.

## Day 4 ✅

### 코딩 트랙

#### 배운 개념

**파일 입출력**
- `open("파일", "w")`: 파일 열어서 쓸 준비 (없으면 생성)
- `open("파일", "r")`: 파일 열어서 읽을 준비
- `open("파일", "a")`: 기존 내용 뒤에 추가
- `.write()`: 파일에 텍스트 기록
- `.read()`: 파일 내용 읽기
- `.close()`: 파일 닫기
- `\n`: 줄바꿈

**JSON**
- 딕셔너리/리스트 구조를 그대로 파일에 저장하고 복원하는 형식
- `import json`: JSON 기능 가져오기
- `json.dump(데이터, 파일, ensure_ascii=False, indent=2)`: 저장
- `json.load(파일)`: 불러오기 (구조 그대로 복원)

**import os**
- `os.path.exists("파일")`: 파일 존재 여부 확인
- 첫 실행 시 파일이 없을 때 에러 방지

**True / False**
- Python 내장 값, "예/아니오"
- 깃발 꽂기 패턴: for문 끝난 뒤 결과를 기억하기 위한 메모지
- `not`: 뒤집기 (not False = True, not True = False)

**= vs ==**
- `=`: 값을 넣어라 (저장)
- `==`: 같은가? (비교, 결과는 True/False)

**데이터 생명주기**
- 불러오기 → 작업(추가/수정/삭제) → 저장
- 불러오기 안 하면 기존 데이터 날아감 (덮어쓰기 방식)
- 변수는 프로그램 실행 중에만 존재, 진짜 데이터는 JSON 파일에 보관

#### 핵심 질문과 이해

**파일 = open()에서 '파일'의 의미**
- Q: "'파일'이라는 말은 왜 써주는 거야?"
- A: 그냥 변수 이름. open()이 돌려준 "파일 연결 통로"를 담아두는 것. 아무 이름이나 가능.

**이름을 붙여두고 명령하기**
- Q: "이름을 붙여두면 그 아랫줄부터 그 대상에만 적용되는 거야?"
- A: 맞음. `파일 = open("메모.txt", "w")` 후 `파일.write()`하면 메모.txt에 쓰임.

**따옴표, 괄호, 중괄호, 대괄호의 차이**
- `" "`: 텍스트 (글자)
- `( )`: 함수 실행 (재료를 넣고 실행)
- `{ }`: 딕셔너리 (항목별 데이터 묶음)
- `[ ]`: 리스트 만들기 / 딕셔너리에서 값 꺼내기

**쉼표의 의미**
- Q: "쉼표로 구분하면 여러 명령을 함수에 담아 실행하는 건가?"
- A: 명령 여러 개가 아니라 "재료(옵션) 여러 개"를 나열하는 것.

**json.load vs 파일.read**
- Q: "왜 파일.read()는 그냥 쓰는데 json.load(파일)은 파일을 지정해야 하나?"
- A: read()는 파일 자체의 기능, json은 외부 도구라서 대상을 알려줘야 함.

**y = f(x) 구조 발견**
- Q: "명령이 전부 y=f(x) 형태네?"
- A: 맞음. 결과 = 함수(재료). Python의 모든 함수가 이 구조.

**왜 환자목록 = 불러오기()인가**
- Q: "불러오기() = 환자목록이 더 자연스럽지 않나?"
- A: `=`는 "오른쪽 결과를 왼쪽에 넣어라"가 고정 규칙. 수학의 y = f(x)에서 온 것.

**변수 충돌 문제**
- Q: "불러오기의 환자목록과 while의 환자목록이 충돌하지 않나?"
- A: 함수 안의 변수는 함수가 끝나면 사라짐. return으로 값만 전달. 같은 이름이어도 시간 순서상 충돌 없음.

**True/False의 정체**
- Q: "False, True가 계속 돌려라는 뜻인가?"
- A: 아님. 그냥 "예/아니오" 값. 반복을 제어하는 게 아니라 for문 끝난 뒤 결과를 기억하기 위한 메모지.
- Q: "False와 True는 Python이 알아듣는 말인가?"
- A: 맞음. Python에 내장된 고정된 값 두 개. 숫자, 글자와 함께 기본 자료형.

**if not의 위치**
- Q: "if not이 for에 종속되어야 하지 않나?"
- A: for 밖에 있어야 함. for 안에 넣으면 매번 확인해서 "없습니다"가 여러 번 출력됨. for 밖에 있어야 "다 돌아본 뒤에 한 번만 확인".

**for문 중 리스트 수정 문제 (직접 발견한 버그)**
- 동명이인 A, B가 각 2명씩 [A, B, A, B] 순서일 때
- A 삭제: 두 명 다 삭제됨 (index 밀림으로 두 번째 A도 삭제)
- B 삭제: 한 명만 삭제됨 (index 밀림으로 두 번째 B 건너뜀)
- 원인: for문은 "몇 번째까지 봤다"만 기억하고, 리스트 변화를 모름
- 해결: break로 즉시 탈출

**break vs 저장하기**
- Q: "break 대신 저장하기를 쓰면 안 되나?"
- A: 안 됨. for문은 메모리의 리스트를 보고 있고, 저장하기는 JSON 파일에 쓸 뿐. 파일에 저장한다고 for문이 멈추거나 리스트를 다시 읽지 않음. for문을 멈추는 건 오직 break.

**f-string 작동 원리**
- Q: "f-string은 틀을 만들어두고 {} 자리에 값을 채우는 건가?"
- A: 맞음. `f` 없으면 글자 그대로, `f` 있으면 {} 안의 변수를 찾아서 값으로 채움.

**콜론(:)의 의미**
- Q: "줄 바꾸면 then이 생략된 건가?"
- A: 맞음. `:`이 then 역할. 다른 언어에서는 실제로 then을 쓰기도 함.

#### 직접 작성한 코드

**삭제 기능** (첫 자작 코드!)
```python
elif 명령 == "삭제":
    검색어 = input(" 삭제할 환자 이름: ")
    찾음 = False
    for 환자 in 환자목록:
        if 환자["이름"] == 검색어:
            환자목록.remove(환자)
            print(f"\n{환자['이름']}환자를 삭제하였습니다.\n")
            print(f"현재 목록에 ({len(환자목록)})명의 환자가 있습니다.")
            찾음 = True
    if not 찾음:
        print("삭제할 환자가 없습니다.")
```
→ 개선점: break 추가 (for문 중 리스트 수정 문제 방지), 저장하기() 추가

**수정 기능**
```python
elif 명령 == "수정":
    검색어 = input(" 수정할 환자 이름: ")
    수축기 = int(input("수정할 혈압: "))
    이완기 = int(input("수정할 혈압: "))
    찾음 = False
    for 환자 in 환자목록:
        if 환자["이름"] == 검색어:
            환자["수축기"] = 수축기
            환자["이완기"] = 이완기
            저장하기(환자목록)
            print(f"\n {환자['이름']} 환자의 혈압을 수정하였습니다. \n")
            찾음 = True
    if not 찾음:
        print("수정할 환자가 없습니다.")
```
→ 개선점: 환자 찾은 뒤에 혈압 입력받는 것이 더 자연스러움 (존재하지 않는 환자에 대해 미리 입력받는 것 방지)

### AI 트랙

**LLM (Large Language Model)**
- "다음에 올 단어를 예측하는 기계"
- 수조 개의 텍스트에서 패턴을 학습
- 진짜 "이해"하는 건 아니고 패턴 매칭 → 할루시네이션 발생 원인

**토큰**
- LLM이 텍스트를 처리하는 단위
- 의미 단위가 아니라 "빈도 단위" — 자주 나오는 조합은 1토큰, 드문 조합은 여러 토큰으로 쪼개짐
- 한글은 영어보다 토큰을 더 많이 사용 → 비용 더 높음
- 비용, 컨텍스트 윈도우 한계, 프롬프트 최적화와 직결

**비용 감각**
- 환자 차트 1건 분석: 약 50~100원
- 하루 30명: 약 1,500~3,000원
- 월: 약 5~10만 원

- 개념 조합 능력: 배운 것을 조합해서 새 기능 제안 (예: input + 딕셔너리 조합)
- 버그 발견 능력: 동명이인 삭제 시 비일관적 동작을 직접 발견하고 원인 추적
