## 프롬프트 튜닝 — 실제 테스트로 약점 발견 → 개선

### 테스트 1: 기본 차트입력
```
입력: BP 135/85, FBS 105, A1c 5.6, LDL 145, HDL 45, TG 180
결과:
  ✅ 활력징후/생활습관 정확 추출
  ✅ 기존참조/신규 정확 구분
  ✅ 검토의견 의학적으로 적절 (공복혈당장애, TG 상승, 추적계획 완료)
  ✅ 의사가 동의/반려 판단 가능
```

### 발견된 문제 1: 날짜 형식
```
문제: "2604" → "260400"으로 변환 (00일은 없음)
해결: YYMM → YYMM01로 채우는 규칙 추가
이후 수정: 추적계획은 YYMM 4자리 그대로 저장하기로 변경 (아래 참조)
```

### 발견된 문제 2: "약 2개월" 해석 오류
```
문제: AI가 "약 2개월" → "2개월 후 추적계획"으로 해석
실제: "오늘 2개월치 약 처방"이라는 뜻
해결: ** 계획 항목 해석 규칙 추가 (날짜 유무로 구분)
```

### 발견된 문제 3: 방문일 기준 계산
```
문제: 이전 차트를 수정하면 오늘 날짜 기준으로 계산됨
실제: 방문일 기준으로 계산해야 함
해결: 차트입력 시 방문일 입력받도록 수정, AI에게 방문일 전달
```

### 발견된 문제 4: 브리핑 시제 오류
```
문제: "이번 방문 확인사항"에서 이전 내용을 오늘 한 것처럼 표현
  → "TSH 1.8로 정상범위 내 잘 조절됨" (이미 확인한 것처럼)
실제: 브리핑은 "오늘 환자를 보기 전"에 읽는 문서
  → "이전 TSH 1.8로 잘 조절되고 있음, 금일 검사 시행하여 확인 예정"
해결: 브리핑 시제 규칙 추가
```

## 차트 스타일 규칙 (SYSTEM_PROMPT에 반영)

### 섹션 구분
```
[섹션1: 진단/문제목록/검사결과]
*
[섹션2: 신체검진/생활습관]
*
[섹션3: 진료내용 — free-text]
*
[섹션4: 계획 — ** 항목들]
```

### 별표 규칙
```
* (1개)  = 섹션 구분자
** (2개) = 계획 항목

날짜 없는 ** → 방문일에 시행/처방한 것:
  **약 2개월        → 처방요약 + 추적계획(재방문)
  **Lab(A1c, Lipid) → 검사결과: 시행일=방문일, 결과=미확인
  **경동맥 초음파    → 영상검사: 시행일=방문일, 결과=미확인

날짜 있는 ** → 해당 월에 할 예정:
  **2604 Lab(A1c)   → 추적계획: 예정일=2604
  **2701 Chest CT   → 추적계획: 예정일=2701
```

### 하위 내용 표시
```
상위 내용
- 하위 내용 (같은 맥락으로 분석)

예:
  흉부 CT 시행.
  - Emphysema 확인 → 하위 (CT 결과)
  - r/o Lipoma → 하위 (CT 결과)
```

### 진단명 표기
```
#진단명 → 진단 추출
#r/o 진단명 → 상태="의심"
#진단명(비고) → 진단명과 비고 분리
```

## 날짜 저장 로직 확정

### 테이블별 날짜 형식
```
검사결과/영상검사 시행일: YYMMDD 6자리 (실제 날짜, 정확)
추적계획 예정일:         YYMM 4자리 허용 ("그 달쯤")
방문기록 방문일:         YYMMDD 6자리 (실제 날짜)
```

### 방문일 기준 계산
```
모든 날짜 계산은 방문일 기준 (오늘 날짜가 아님!)
main_system.py에서 방문일 입력받도록 수정:
  방문일 = input("방문일 (YYMMDD, 오늘이면 Enter): ")
chart_analyzer.py에 방문일 매개변수 추가
```

### 추적계획 완료 매칭
```
추적계획(YYMM)과 검사결과(YYMMDD)는 형식이 다름
→ 날짜로 매칭하지 않고 내용(검사항목)으로 매칭

예: 추적 "2604 Lab(A1c)" + 검사 "260510 A1c" → 내용 일치 → 완료 처리
환자가 예정일과 다른 날 와도 문제없음!
```

### 통계 시 주의점
```
단일 테이블 내 정렬/검색/통계 → 문제없음
테이블 간 날짜 비교 (예: 예정일 vs 실제일 차이) → YYMM vs YYMMDD 비교 불가
→ 필요 시 Week 4에서 추적계획에 '실제시행일' 칸 추가 예정
```

## 브리핑 형식 확정

### 형식
```
[진단/문제목록] — 활성/의심 진단 나열
[검사결과 추이] — 항목별 수치 변화 (화살표)
[임상 요약] — 한 문단 서술 (이전 경과 + 오늘 목적)
[이번 방문 확인사항] — 진단별 문제 중심 접근
  진단명
  * 이전 상태 (과거 표현)
  * 오늘 할 일 (미래/현재 표현)
  금일 핵심 업무 한 문장 요약.
```

### 시제 규칙 (매우 중요)
```
브리핑 = "오늘 환자를 보기 전"에 읽는 문서

이전 기록 → '이전', '지난 방문', '기존' (과거 표현)
오늘 할 일 → '금일', '오늘', '이번 방문' (미래/현재 표현)
마지막 요약 → '오늘 OOO해서 OOO 확인해야 함' (행동 지시형)
```

## 학습 과정 관찰

### 프롬프트 튜닝의 본질
```
테스트 → 문제 발견 → 원인 분석 → 프롬프트 수정 → 재테스트

이 과정에서 선생님의 의학적 판단이 핵심:
  - "약 2개월은 처방이지 추적이 아니야"
  - "날짜 없으면 오늘 한다는 뜻이야"
  - "브리핑은 환자 보기 전에 읽는 거야"
  - "추적 예정일은 '그 달쯤'이라는 의미야"
  → 전부 의사만 알 수 있는 도메인 지식!
```

### 설계 능력의 성장
```
Day 8:   JSON vs DB 차이 이해
```
