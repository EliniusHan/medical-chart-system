## 문제 발견 및 해결

### 1. 미측정 값 0으로 저장 → NULL로 변경
```
문제: BP 안 쓰면 0으로 저장 → 통계 시 평균 왜곡
해결: 0 대신 NULL 저장, 출력 시 "미측정" 표시
용어: "널러블(nullable) 필드" — 값이 없을 수 있는 칸
```

### 2. 신환등록 간소화
```
문제: 등록 시 가족력/약부작용을 물어보지만, 그 시점에 정보가 없을 수 있음
해결: 등록은 이름/생년월일/성별만, 가족력/약부작용은 차트입력 시 AI가 감지
```

### 3. 가족력/약부작용 — 누적/수정/삭제 처리
```
추가: "어머니한테 당뇨도 있대요" → 기존 + 새 항목
수정: "뇌경색이 아니라 뇌출혈이었대요" → 기존 내용 수정
삭제: "진통제 부작용은 괜찮았어요" → 의사에게 확인 후 처리

AI가 환자 테이블의 최신 정보와 차트 내용을 비교:
  차트에 옛날 정보 쓰면 → 최신으로 자동 수정 (불일치 감지)
  과거 차트 → 그때 기록 그대로 유지 (의무기록 보존)
```

### 4. AI 보완 차트 기능 → Week 5~6으로 이관
```
문제: AI가 추가한 내용이 차트와 분리되어 표시됨
원하는 것: AI가 의사 스타일로 보완한 차트를 보여주고, 승인 후 저장
이유: 터미널에서는 비교/편집이 어려움 → 웹앱(Week 5~6)에서 구현
```

## 설계 심화 — 날짜 저장 로직

### 추적계획 예정일 YYMM vs 검사결과 시행일 YYMMDD
```
문제: "2604"를 "260401"로 채우면 진짜 1일인지 "그 달쯤"인지 구분 불가
해결: 추적계획은 YYMM 4자리, 검사결과는 YYMMDD 6자리
     매칭은 날짜가 아닌 내용(검사항목)으로

통계 시:
  단일 테이블 내 → 문제없음
  예정일 vs 실제일 비교 → YYMM vs YYMMDD 직접 비교 불가
  → 필요 시 Week 4에서 추적계획에 '실제시행일' 칸 추가
```

### 방문일 기준 계산
```
모든 날짜 계산 = 방문일 기준 (오늘 날짜가 아님!)
이전 차트 수정 시에도 방문일 기준으로 정확히 계산
→ 기술 용어: "컨텍스트 기반 날짜 처리(context-aware date handling)"
```

## 프롬프트 추가 규칙들

### 생활습관 해석 규칙
```
빈칸 = 이번에 안 물어봄 (누락이 아님!)
"비흡연" = 확인함, 해당 없음
"하루 10개피" = 현재 상태
"1갑 → 반갑" = 변경 기록 (최신=반갑)
→ 빈칸일 때 AI가 "누락됐다"고 잘못 지적하지 않도록
```

### 가족력/약부작용 감지 규칙
```
AI가 free-text에서 감지 → 변경유형(add/modify/remove) 판단
→ 의사에게 확인: "가족력 업데이트할까요?"
→ 승인 시 환자 테이블 업데이트

환자 테이블 ≠ 차트 내용 시:
→ 환자 테이블이 최신 → AI가 차트 내용 자동 수정 제안
```
