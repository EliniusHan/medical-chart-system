## 진행 내용

### 코드 리딩 + 복습 퀴즈

1. **copy.deepcopy()**: 완전히 별도 복사본 생성. 복사본 수정해도 원본 안 바뀜. 익명화 시 원본 환자데이터 보존에 필수.
2. **분석완료 버그 복습**: 방문기록수정()이 호출될 때마다 새 방문id 생성 → 원래 id는 유효여부=0 → 분석완료처리(원래id)가 효과 없음. 해결: 방문기록_일괄수정() + 환자id+방문일 기준 분석완료처리.
3. **방문기록_일괄수정()**: 허용필드(안전장치)로 수정 가능한 칼럼 제한. 기존 방문기록수정()(정정 방식, 새 레코드 생성)과 달리 단순 UPDATE(레코드 복사 안 함). AI 최초 저장용.

## 핵심 개념 정리

### SQL 체크 vs AI 분석
```
SQL 체크 (데일리_SQL체크):
  → 날짜 비교로 "늦었다/밀렸다" 판단
  → 매일, 무료, 즉시 실행
  
AI 분석 (AI_패턴분석):
  → 의학적 판단 "잘못됐다/패턴 이상"
  → 주/월 1회, 유료, API 호출
```

### 방문기록수정 vs 방문기록_일괄수정
```
방문기록수정():
  → 의사 수동 정정용
  → 원본 보존 + 새 레코드 생성 (의무기록 원칙)
  → 호출마다 방문id 변경

방문기록_일괄수정():
  → AI 최초 저장용
  → 단순 UPDATE (새 레코드 안 만듦)
  → 여러 필드 한 번에 수정
  → 방문id 안 바뀜
```
