## 연구 데이터 모듈 구축

### research_module.py 완성
```
3가지 기능:
  1. 자연어 검색 (NL2SQL) — 의사가 질문 → AI가 SQL 변환 → 결과 추출
  2. AI 자동 통계 분석 (모드 1) — 질문 → AI가 전부 알아서 처리
  3. 단계별 통계 분석 (모드 2) — 질문 → AI가 제안 → 의사가 단계별 확인/수정
```

### 자연어 검색 (NL2SQL) 테스트 결과
```
✅ 단순 검색: "전체 환자 목록"
✅ 조건 검색: "고혈압 환자 중 LDL 100 이상"
✅ 다중 조건: "고혈압이면서 흡연하는 환자"
✅ 시간 기반: "260301 이후 방문"
✅ 추적계획: "미완료 추적계획 있는 환자"
✅ free-text: "흉부외과 의뢰한 환자"
✅ cross-table JOIN: "고혈압 + 경동맥초음파 + LDL"
✅ 빈 결과 처리: "해당 없음" 안내
✅ CSV 저장 옵션

기술 용어: NL2SQL (Natural Language to SQL) — AI 분야에서 핫한 주제
```

### JOIN 시 중복 행 발견
```
김영수가 LDL 검사 2건이라 결과가 2행으로 나옴
→ 기술 용어: "카디널리티 폭발(cardinality explosion)"
→ JOIN 시 검사 건수만큼 행이 늘어나는 현상
→ 나중에 "가장 최근 검사만" 가져오도록 프롬프트 개선 가능
```

## 통계 분석 기능

### 모드 1: AI 자동 분석
```
흐름:
  의사: "고혈압 환자들의 평균 혈압과 LDL 분석해줘"
    ↓
  1단계: AI가 SQL + 통계 코드 생성 → 실행 → 결과 확보
  2단계: 실제 결과를 AI에게 다시 보내서 해석 요청 (할루시네이션 방지!)
    ↓
  결과 + 그래프 + 분석 과정 설명
```

### 모드 2: 의사가 단계별 선택
```
흐름:
  Step 0: 연구 질문 정의 (핵심!)
  Step 1: 데이터 선택 (AI 제안 → 의사 승인/수정)
  Step 2: 그룹 설정 (AI 제안 → 의사 승인/수정)
  Step 3: 통계 기법 선택 (AI 추천 or 직접 선택)
  Step 4: 결과 표시 + 실제 결과 기반 해석
  Step 5: 그래프 저장
  Step 6: CSV 내보내기

각 단계에서 수정 요청 시:
  의사가 free-text로 수정사항 입력 → AI가 재제안 → 다시 확인 → 반복
```

## 할루시네이션 발견 및 해결

### 발견: 분석 결과와 해석의 숫자 불일치
```
실제 분석: 수축기 148.0±12.2, LDL 147.5±18.9, r=0.112, p=0.792
결과 해석: 수축기 약 145±18, LDL 약 125±35, r=0.234, p=0.045
→ AI가 실제 결과를 보지 않고 숫자를 지어냄!
→ 의학 연구에서 매우 위험

기술 용어: "출력 검증(output validation)" — AI 생성 숫자가 실제 데이터와 맞는지 확인
```

### 해결: 2단계 분리
```
1단계: AI가 코드 생성 → 실행 → 실제 결과 숫자 확보
2단계: 실제 결과를 AI에게 다시 보내서 해석 요청
→ "이 숫자를 기반으로 해석하세요. 절대 변경하지 마세요."
→ 해석의 숫자가 실제 결과와 반드시 일치!
```

## 통계 오류 처리

### 에러 메시지 — 친절한 설명 + 전문 용어 함께 표시
```
알려진 패턴 (상세 설명):
  Singular matrix — 표본 부족/공선성 + 풀어쓴 설명
  ConstantInputWarning — 변수 분산 0 + 풀어쓴 설명
  Shapiro-Wilk — 정규성 위반 + 대안 제시
  Levene's test — 등분산성 위반 + 대안 제시
  Convergence failure — 수렴 실패 + 대안 제시

모르는 패턴:
  → AI에게 실시간 해석 요청 + 대안 제시

모드 1, 2 동일하게 적용 (공통 함수 통계오류처리())

기술 용어: "그레이스풀 디그레이데이션(graceful degradation)"
  → 에러가 나도 시스템이 우아하게 대응하는 것
```
