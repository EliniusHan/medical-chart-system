## 진행 내용

### 연구 모듈 전체 테스트 + 버그 수정

#### 1. 자연어 검색 (NL2SQL) ✅
- 테스트: "고혈압 환자 중 LDL이 100 이상인 환자 목록"
- 결과: 7건 정확 일치 (DB 직접 검증 완료)
- 발견: SQL에 유효여부=1 필터 누락 → 프롬프트 강화로 해결

#### 2. AI 자동 통계 분석 ✅
- 테스트: "고혈압 환자와 비고혈압 환자의 평균 수축기 혈압 비교"
- 결과: 고혈압군 6명 148.7±7.3 vs 비고혈압군 44명 116.1±4.6 (p<0.001)
- DB 직접 검증 완료, 정확 일치
- 복잡한 분석(전후 비교) 실패 → 에러 표시 + 대안 제시로 개선

#### 3. 단계별 통계 분석 ✅
- 테스트: "당뇨 확진 환자 A1c 분포" → 기술통계 + 히스토그램
- 결과: 12명, 평균 8.13%, 중앙값 8.00% — DB 직접 검증 완료
- 그래프 생성 + 성별/연령대별 분석 포함

### 발견 및 수정한 버그들

#### 🔴 Step 4 JSON 파싱 실패 (근본 원인)
- 원인: Step 4에서 STEP_SYSTEM을 사용 → "한국어로 간결하게 답변" 지시만 있고 JSON 형식 규칙 없음 → AI가 텍스트로 응답 → 파싱 실패 → code="" → exec("") → "(결과 없음)"
- 해결: Step 4에서 STAT_SYSTEM 사용으로 변경 (JSON 형식 + 코드 생성 규칙 포함)

#### 🔴 유효여부 필터 과적용
- 원인: "모든 테이블에 유효여부=1" 강조 → AI가 환자 테이블에도 적용 → "no such column: p.유효여부"
- 해결: "환자 테이블에는 유효여부 칼럼이 없음!" 명시. 유효여부가 있는 7개 테이블 목록 제공

#### 🔴 그래프 파일명 충돌
- 원인: 초 단위 타임스탬프만 사용 → 같은 초에 실행하면 덮어쓰기
- 해결: uuid.uuid4().hex[:4] 추가 (예: analysis_20260412_012507_40b9.png)

#### 🔴 이전 그래프 폴백
- 원인: 새 그래프 생성 실패 시 폴더 내 최신 파일을 찾아서 반환 → 이전 분석 그래프가 표시됨
- 해결: 폴백 로직 제거. graph_path에 파일이 있을 때만 반환

#### ⚠ Step 3 UX 루프
- 원인: AI 제안에 y 해도 하드코딩 목록이 또 나옴
- 해결: 하드코딩 목록은 직접선택(m) 시에만 표시

#### ⚠ Step 2 하드코딩 목록 혼란
- 원인: AI가 "1.A1c별 2.연령대별" 제안 → 그 아래에 "1.두 그룹 비교 2.전후 비교" 또 나옴
- 해결: Step 2에서 하드코딩 목록 제거. AI 제안 번호로만 선택

#### ⚠ Step 3 상황별 질문 분기
- _복수방법_감지()로 AI 제안에 번호가 2개 이상이면 복수 방법으로 판단
- 복수 방법: 번호 선택(하나) / 조합(여러 개) / AI에게 맡기기(y) / 직접선택(m)
- 단일 방법: AI 추천대로 진행(y) / 직접선택(m)

### 핵심 기능 요약
```
1. 차트 분석 (API 1~2회)
   의사 free-text → AI 제안(💡⚖설명의무) → 승인 → free-text 보완
   → 데이터 추출 → 항목별 승인 → 8개 테이블 저장
   + 공공 DB 연동 (DUR 병용금기, PubMed 논문, 가이드라인 출처)
   + 의사 패턴 맥락 (의사패턴_요약생성)

2. 환자 브리핑
   문제 중심 브리핑 + AI 권고사항 (미시행 검사, 미이행 계획, 약물 모니터링)

3. 연구 모듈
   NL2SQL / AI 자동 통계 / 단계별 통계 (상황별 UX)
   + free_text AI 분석

4. 진료 패턴 분석
   데일리 SQL 체크 (무료) / AI 패턴 분석 (유료)
   환자 목록 포함 문제점 표시

5. 보안
   익명화 (이름→PT_XXXXX, 생년월일→나이대)
   의무기록 보존 (유효여부 플래그, 정정 이력)
   API 키 분리 (.env)
   백업 (타임스탬프, 최근 5개 유지)
```
