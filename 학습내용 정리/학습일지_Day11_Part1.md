### 차트입력 + 검토 합치기 (API 1번으로 통합)
```
기존:  추출 API 1번 + 검토 API 1번 = 2번
개선:  추출 + 검토를 한 프롬프트로 = 1번
→ 비용 절감 + 시간 절감 + 확인도 1번
```

### 검사결과 TEXT vs 숫자 통계 문제
```
현재: 결과값이 TEXT ("124", "C1 negative" 혼재)
문제: SQL에서 AVG() 등 통계 함수 사용 불가
해결: Week 4에서 결과수치(REAL) 칸 추가 예정
당장: CAST() 변환이나 AI 분석으로 대체 가능
```

### 진단 상태 변경
```
기존: 활성, 관해, 종결
변경: 활성, 의심, 종결 (관해 제거 — 모호하므로)
세부사항은 진단명 자체에 포함: #고혈압(LSM), #고혈압(치료 안 함)
```

### 진단명 파싱 (AI 역할)
```
의사가 "#고혈압(LSM)" → AI가 분리 → 진단명="고혈압", 비고="LSM"
의사가 "#r/o 고혈압" → AI가 분리 → 진단명="고혈압", 상태="의심"
의사 습관 안 바꿔도 됨 — AI가 알아서 정리
```

### 표준코드(KCD/ICD) 도입
```
진단 테이블에 표준코드 칸 추가 완료
매핑 가능한 진단: 표준코드="I10"
매핑 불가: 표준코드=NULL
연구 시: 코드 있음 → 자동 포함, NULL → AI가 후보 추려서 의사 판단
```

## 핵심 질문과 이해

### API 관련
**"기존 API 키를 다른 앱에서 쓰고 있는데 상관없나?"**
- 상관없음. API 키는 인증일 뿐, 앱 구분 없이 사용 가능
- 비용은 키 단위가 아니라 계정 단위로 청구

**"크레딧 오류가 나는데?"**
- API 크레딧은 claude.ai 구독과 별도 → 따로 충전 필요
- $5면 수백 회 테스트 가능

### 시스템 설계 관련
**"경과기록작성과 차트입력이 겹치지 않나?"**
- 차트입력(AI)이 경과기록작성(수동)을 완전히 포함
- 경과기록작성은 admin 히든 메뉴로 백업용 유지

**"모든 항목을 다 물어보면 비효율적이지 않나?"**
- 지금은 기능 확인용 → 나중에 전체승인+수정 → 최종적으로 자동승인 단계적 개선

**"분석+검토를 합치면 일을 두 번 안 해도 되지?"**
- 맞음. 1번의 API 호출로 추출+검토 동시 처리 → 확인도 1번 → 저장도 1번

**"AI 실패하면 어떡하지?"**
- free_text만 저장(분석완료=0) → 다음 로그인 시 자동 체크 → AI 복구 후 분석

**"결과값이 TEXT면 통계 못 돌리지 않나?"**
- Week 4에서 결과수치(REAL) 칸 추가 예정

**"진단 상태에서 관해가 모호해"**
- 제거. 활성/의심/종결 3개로 변경. 세부사항은 진단명에 포함.

**"의사마다 진단 표현이 다르면?"**
- 표준코드(KCD/ICD) 매핑 + AI가 진단명+비고 분리 → Week 7~8에서 적용
