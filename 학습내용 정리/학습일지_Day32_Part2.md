- **수정**: 헬퍼로 양쪽 날짜 모두 YYMMDD로 정규화 후 비교.
- **검증**: 추후 수행 예정. (오류 1 수정과 다른 세션이라 미검증)

- **파일**: `app.py` (_chart_step1)

- **검증 상태**: 보류. API 키 일부러 망가뜨리는 시나리오라 지금 안 함.

#### ③ 미리보기 대표값 검사명/검사종류 누락 [검증 완료]

- **검증 결과**: 처음엔 "검사결과"로 잘못 분류되는 버그가 노출됨(오류 1로 이어짐). 오류 1 수정 후 정상 동작.

#### ④ 재분석 저장에 사용자 정정사유 전달 [적용 완료, 검증 보류]

### 2차 수정 (오류 1 + 가상 방문 설계 확장) [적용 완료, 검증 보류]

#### 오류 1: `_변경사항_추출` 프롬프트 보강
- **파일**: `chart_analyzer.py` (_변경사항_추출 내부 시스템 프롬프트)
- **문제**: 기록 수정/삭제 탭에서 `**CXR`, `**경동맥 초음파` 같이 결과 수치 없는 검사를 추가하면 검사처방이 아닌 검사결과로 잘못 분류됨. 원인은 이 함수의 프롬프트가 `_EXTRACTION_RULES`와 별도라서 "검사 분류 최우선 원칙"이 없었음.
- **수정**: 프롬프트에 다음 추가:
  - 검사 분류 최우선 원칙 (결과 수치/소견 → 검사결과·영상검사 / 없으면 → 검사처방)
  - `**검사명` / `**영상명` 표기법 해석 규칙
  - 5개 테이블(검사결과/검사처방/영상검사/처방/진단/추적계획) JSON 예시
  - 날짜 변환 규칙, 단위/참고범위 자동 채우기 규칙

#### 수정 B: 검사결과/영상검사 저장 시 방문id 자동 결정
- **파일**: `chart_analyzer.py`
- **추가된 헬퍼**: `_방문id_결정(환자id, 검사시행일, 현재방문id)`
  - (a) 검사시행일 없음 → 현재방문id
  - (b) 시행일 == 현재 방문일 → 현재방문id
  - (c) 같은 시행일의 유효 방문 존재 → 그 방문id
  - (d) 없으면 가상 방문 레코드 생성 (free_text='', 분석완료=1) → 그 방문id
- **호출부 수정 6곳**: `분석결과_저장()` 2곳, `차트_데이터만_수정()` 4곳 (변경/추가 × 검사결과/영상검사)
- **의도**: 시나리오 1(외부 기록)에선 가상 방문 생성, 시나리오 2(과거 우리 병원 방문)에선 기존 방문 연결. 모두 검사결과의 방문id가 "시행 당시 맥락"을 가리키도록.

### 3차 수정 (가상 방문 UI 숨김 + AI 호출 분리) [적용 완료, 검증 예정]

가상 방문 도입 후 필요해진 후속 정리. 다음 두 원칙 적용:
- **UI**: 가상 방문은 전체이력 탭 버킷 + 수정/삭제 탭 목록에서 숨김
- **AI 호출**: 진료 권고 생성용(브리핑, 패턴분석) 호출에선 외부 기록 제외.
  차트 작성/수정용 호출(차트분석, 재추출, _변경사항_추출)에선 외부 기록 포함.
  - 이유: 의료 책임 원칙. 외부 병원 검사 결과를 근거로 AI가 권고 주면 안 됨.
  - 차트 작성 중엔 의사가 외부 기록을 차트에 쓸 때 AI가 맥락을 이해해야 함 → 포함 유지.

#### util.py
- `_가상방문_SQL조건` 상수 추가 (SQL WHERE 절에서 쓰는 판별식)
- `_가상방문_여부(방문_dict)` Python 헬퍼 추가
- `환자전체기록조회()`에 `외부기록_제외=False` 파라미터 추가
  - True 시 가상 방문 및 그에 연결된 검사결과/영상검사 제거

#### app.py
- import에 `_가상방문_여부` 추가
- 전체이력 탭: 가상 방문 버킷 생성 안 함 + 가상 방문에 연결된 검사결과/영상검사 표시 안 함
- 기록 수정/삭제 탭: 방문 목록에서 가상 방문 숨김 + 가상 방문 편집 진입 차단

#### briefing_generator.py
- `브리핑생성()`, `브리핑생성_최근차트()` 모두 `외부기록_제외=True` 전달

#### practice_analyzer.py
- import에 `_가상방문_SQL조건` 추가
- `_DB_요약수집()`의 검사결과 조회 쿼리에 가상 방문 제외 조건 추가
- `_환자별_상세수집()`의 `MAX(방문id)` 서브쿼리에 가상 방문 제외 조건 추가
  - **중요**: 이게 없으면 가상 방문의 방문id가 더 커서 MAX를 차지하고, 가상 방문엔 처방이 없어서 "현재 처방"이 빈 리스트로 나옴. 가상 방문 도입으로 생긴 신규 버그.

## 가상 방문 식별 조건 (확정)

방문 레코드가 다음 조건을 모두 만족하면 가상 방문으로 간주:
- `free_text IS NULL 또는 ''`
- 수축기, 이완기, 심박수, 키, 몸무게, BMI 모두 NULL
- `분석완료 = 1`
- `유효여부 = 1`

기존 DB에 이 조건을 만족하는 기존 레코드 없음 확인(오탐 방지 검증 완료).

- DB 조회로 정정이력 확인. 무효화된 레코드 정정사유가 `"검사수치 오기재 검증 (재분석에 의한 무효화)"` 형식인지.

**2. 오류 1 (검사처방 분류) 재검증**
- 기존 차트 free-text에 `**CXR`, `**경동맥 초음파` 추가 → 변경사항 확인
- 기대: `➕ 검사처방 — CXR 추가`, `➕ 검사처방 — 경동맥 초음파 추가`

**3. 시나리오 1 (외부 기록) — 가상 방문 생성 확인**
- 오늘 차트 만들어서 저장 후, 수정 탭에서 `250820 A1c 5.7` 추가 → 저장
- DB 확인:
  ```sql
  -- 방문 테이블
  방문일=250820, free_text='', 분석완료=1, 유효여부=1 인 레코드 생성됐는지
  -- 검사결과 테이블
  방문id가 위 가상 방문을 가리키는지, 검사시행일=250820, A1c 5.7 인지
  ```

**4. 시나리오 2 (과거 우리 방문 존재) — 기존 방문 연결 확인**
- 기존 환자 중 과거 방문(예: 260301)이 있는 케이스 선택
- 오늘 차트에 "260301 A1c 7.5" 기재 → 저장
- 기대: 새 방문 생성 안 됨. 검사결과의 방문id가 기존 260301 방문id

**5. UI 숨김 확인**
- 전체이력 탭: 250820 버킷 안 보임. 오늘 버킷 안의 검사결과 섹션엔 아무것도 없음 (free_text에만 원문).
- 기록 수정/삭제 탭: 250820 가상 방문이 목록에 안 나타남.

**6. 브리핑 외부 기록 제외 확인**
- 브리핑 탭 → 최근 차트 브리핑 생성
- 기대: AI 응답에 외부 검사 결과 언급 없음. 우리 병원 진료 내용만 기반.

**7. AI 패턴 분석 MAX(방문id) 보정 확인**
- 홈의 AI 패턴 분석 실행
- 가상 방문이 있는 환자의 "현재 처방"이 빈 리스트로 나오지 않아야 함.

**8. ② Step 1 orphan 검증** (API 키 일부러 망가뜨리는 시나리오, 여유 있을 때)

## 알고 있는 한계 / 나중에 처리할 것

- **방문일 변경 기능 없음**: Step 1에서 방문일을 잘못 입력한 뒤 발견해도 수정 불가 (방문기록_일괄수정의 허용필드에 방문일 없음). 방문 삭제 후 재생성해야 함.
- **가상 방문 내부 검사 여러 건 누적**: 같은 외부 날짜(예: 250820)에 A1c + TG + LDL 등 여러 검사가 시간차로 추가될 수 있음. 이땐 동일한 가상 방문에 누적됨. 의도된 동작.
- **가상 방문 "승격" 금지**: 외부 기록이 들어간 가상 방문에 의사가 나중에 free_text를 채워넣는 흐름은 지원 안 함 (병원 실무상 있어서는 안 되는 일).
- **practice_analyzer**: AI 패턴분석이 외부 기록 제외하지만 데일리_SQL체크는 그대로. 추적 지연/미시행 체크 등은 운영 지표라 가상 방문 포함이 딱히 해롭지 않음. 필요 시 추후 수정.
