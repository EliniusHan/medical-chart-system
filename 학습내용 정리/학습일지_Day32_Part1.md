## Day 32 완료한 것

### 백엔드 개선
- **검사 분류 버그 수정**: _EXTRACTION_RULES 모순 해결 (lab_results vs test_orders)
- **Lab() 표기법**: "Lab(A1c, Lipid)" → 기본패널 + 괄호 항목 통합 test_orders
- **검사처방 시행완료 자동 매칭**: 정확한 날짜만 비교 (같은 달 매칭 제거)
- **검사결과 단위/참고범위 자동 채우기**: _검사_참고정보 딕셔너리 (util.py)
  - 대소문자/공백 무시 매칭
  - 성별에 따라 다른 참고범위 적용
- **날짜 정규화**: 4자리 YYMM, 6자리 YYMMDD, 10자리 YYYY-MM-DD 모두 처리
- **_변경사항_추출() 프롬프트**: 날짜 변환 규칙 + 단위/참고범위 자동 채우기 규칙 추가
- **방문기록_일괄수정 허용필드에 "분석완료" 추가**
- **방문기록추가 기본값 분석완료=0으로 변경** (수동입력도 미분석으로 기록)

### 프론트 개선
- **진료기록 탭 5단계 전체 구현**: Step 1 입력 → 2 제안 → 3 보완 → 4 추출 → 5 저장
- **기록 수정/삭제 탭 통합**: free-text 기반 수정, 방문 삭제, 환자 삭제
- **AI 재분석 Y/N**: 당일 차트만 재분석 가능, 과거 기록은 데이터 수정만
- **정정사유 입력**: 과거 기록 수정 시에만 필수
- **변경사항 미리보기**: [변경사항 확인] → AI 추출 결과 표시 → [확정 저장]
- **API 이중 호출 방지**: 미리보기 결과를 저장 함수에 전달
- **탭 이름**: "기록 수정" → "기록 수정/삭제"
- **목록으로 버튼**: 제목 오른쪽에 배치
- **환자 메모 기능 추가**:
  - 환자 테이블에 메모 칼럼 (DEFAULT '')
  - 환자메모_조회(), 환자메모_저장() 함수
  - AI/통계에 반영되지 않음 (anonymizer에서 제거)
  - Step 1에서 진료기록 옆 우측에 병렬 배치 (7:3 비율)
- **환자 헤더 가족력/약부작용이력**: 상하 → 좌우 병렬 배치

### 버그 수정
- _검사처방_시행완료_매칭: 날짜 무시하고 검사명만 매칭하던 버그 → 정확한 날짜 매칭
- 차트_재분석_저장: input() 제거 (Streamlit 호환)
- 기록수정 탭 저장 기능: 변경사항 있을 때만 표시되던 버그 수정

## CLAUDE.md 데이터 수정/삭제 규칙
- 프론트(Streamlit app.py)에서의 수정/삭제는 반드시 free-text 기반으로만 수행
  - 차트_데이터만_수정(): AI가 변경분 감지 → 테이블 업데이트 (재분석 없음)
  - 차트_재분석_저장(): AI 재분석 → 테이블 재생성 (당일 진료 중 수정)
  - 방문기록삭제(): 방문 전체 삭제
  - 환자삭제(): 환자 전체 삭제
- 개별 테이블 수정/삭제 함수는 util.py에 유지하되 app.py에서 호출 금지
