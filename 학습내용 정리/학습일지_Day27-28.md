- 검사처방 테이블: 처방검사id, 환자id, 방문id, 검사명, 처방일, 시행여부, 유효여부, 정정사유
- util.py에 CRUD 함수 추가 (처방추가/조회/수정/삭제, 검사처방 동일 + 시행완료)
- 환자전체기록조회(), 환자삭제(), 방문기록삭제()에 새 테이블 반영

#### 날짜 규칙 수정 (YYMM → YYMM00)
- YYMM(4자리) → YYMM00 저장 (01이 아니라 00!)
- 00 = "이 달 중 언제인지 모름"
- YYMMDD(6자리) = 정확한 날짜 그대로

#### ** 계획 분류 규칙
- YYMM + 검사 → 검사처방 ✅ + 추적계획 ✅ (내원 포함)
- YYMM + 약 → 처방(예정) ✅ + 추적계획 ✅ (내원 포함)
- YYMM + 재방문 → 추적계획만 ✅
- YYMM 없이 "약 N개월" → 처방만 ✅, 추적계획 ❌ (오늘 처방한 것)

#### 법적 보호 기능
- ⚖ 필수 확인/권고사항: 법적으로 반드시 확인/권고해야 하는 것 + chart_text 문구 제안
- 설명의무 기록: 처방 약물 부작용 자동 나열 + 설명 문구 제안
- 둘 다 승인 시 free_text에 추가 (y/n)

#### 차트 분석 흐름 전면 재설계 (가장 큰 변경!)
**새로운 흐름 (API 1~2회):**
1. 의사가 free-text 작성
2. AI 분석 (1번째 API) — 데이터 추출 + 제안 전부 수행. 추출 데이터는 메모리 보관, 제안만 표시
3. 의사가 제안(💡⚖설명의무) 승인/반려 (y/n만, 편집 옵션 제거)
4. 승인된 chart_text를 free-text 아래에 붙여넣기 (함수, API 아님)
5. 의사가 전체 free-text 직접 수정 → 최종 확정
6. 미리 추출해둔 데이터 한꺼번에 보여주고 "AI로 재분석할까요? (y/n)"
   - n: 미리 추출한 데이터로 항목별 승인 → 저장 (API 1회로 끝!)
   - y: 최종 free-text로 재분석 (2번째 API) → 재추출 데이터로 승인 → 저장
7. 승인된 항목만 각 테이블에 저장 → 최종 free-text와 테이블 항상 일치!

#### AI 검토 의견 중복 제거
- 의사가 이미 차트에 언급/인지한 내용은 제안에서 제외
- 놓친 것만 제안

#### 처방 "유지"도 저장
