## 프로젝트 개요

이 프로젝트는 Python을 단계별로 학습하면서 **의료 환자 관리 시스템**을 점진적으로 구축해 나가는 학습용 프로젝트입니다. Day1부터 Day4까지 진행되며, 변수·조건문 같은 기초 문법에서 출발해 파일 입출력과 모듈화까지 확장됩니다. 전체적으로 환자의 이름, 나이, 진단명, 혈압 정보를 다루며, 혈압 판정 로직이 핵심 기능으로 반복 등장합니다.

---

## 파일별 상세 분석

### Day1_step1.py — 변수와 조건문 기초

Python의 가장 기본적인 문법을 익히는 첫 번째 파일입니다. 환자 한 명(김철수)의 정보를 변수에 저장하고 `print`로 출력한 뒤, `if/else` 조건문을 사용해 수축기 혈압이 140 이상인지 판단합니다. 주석을 통해 각 문법 요소를 설명하고 있어 학습 노트 성격이 강합니다.

**사용된 개념:** 변수, print, if/else 조건문

### Day1_step2.py — 딕셔너리, 리스트, 반복문

Step1에서 배운 내용을 확장하여 딕셔너리(환자 차트)와 리스트(환자 목록)를 도입합니다. 두 명의 환자 데이터를 딕셔너리로 만들고, 리스트에 담은 뒤 `for` 반복문으로 순회하면서 각 환자를 브리핑합니다. 혈압 판정도 `elif`를 추가해 고혈압 전단계까지 구분합니다.

**사용된 개념:** 딕셔너리, 리스트, for 반복문, if/elif/else

### Day2.py — 사용자 입력(input) 처리

`input()` 함수를 사용해 의사가 직접 환자 정보를 입력할 수 있도록 합니다. 입력받은 값을 `int()`로 변환하고, 혈압 판정 로직을 5단계(고혈압 응급 → 2기 고혈압 → 1기 고혈압 → 혈압 상승 주의 → 정상)로 세분화했습니다. 이완기 혈압도 판정 기준에 포함시켜 보다 정밀한 판정이 가능합니다.

**사용된 개념:** input, int 변환, 다단계 조건문(if/elif/else)

### Day3_step1.py — 함수(def) 정의와 활용

반복되는 코드를 `def`로 함수화하는 단계입니다. `혈압판정()` 함수와 `환자브리핑()` 함수를 정의하고, 4명의 환자 데이터를 리스트에 담아 반복문으로 브리핑합니다. 코드의 재사용성과 가독성이 크게 향상되었으며, 함수의 `return`과 `print`의 차이를 학습할 수 있는 구조입니다.

**사용된 개념:** 함수 정의(def), return, 함수 호출, for 반복문

### Day3_step2.py — while 반복문과 동적 환자 등록

`while True`와 `break`를 사용한 무한 반복 구조로 환자를 계속 등록할 수 있는 시스템을 구현합니다. 사용자가 이름에 "끝"을 입력하면 등록이 종료되고, `enumerate`를 활용해 환자 번호를 붙여 전체 브리핑을 출력합니다. Day3_step1의 함수를 그대로 재활용합니다.

**사용된 개념:** while True/break, list.append, enumerate, 함수 재사용

### Day4_step1.py — 파일 입출력 기초 (텍스트)

`open()`, `write()`, `read()`, `close()`를 사용해 텍스트 파일(.txt)에 환자 정보를 저장하고 다시 읽어오는 기본적인 파일 입출력을 학습합니다. "w"(쓰기) 모드와 "r"(읽기) 모드의 차이를 주석으로 설명하고 있습니다.

**사용된 개념:** open, write, read, close, 파일 모드(w/r)

### Day4_step2.py — JSON 파일 입출력

`json` 모듈의 `dump`와 `load`를 사용해 딕셔너리 리스트를 JSON 파일로 저장하고 불러옵니다. `ensure_ascii=False`로 한글 깨짐을 방지하고, `indent=2`로 가독성을 높이는 옵션도 학습합니다. 텍스트 파일 대비 구조화된 데이터를 다루기에 적합한 방식임을 보여줍니다.

**사용된 개념:** import json, json.dump, json.load, ensure_ascii, indent

### Day4_step3.py — 종합 환자 관리 시스템 (메인 프로그램)

프로젝트의 최종 결과물입니다. 등록, 삭제, 수정, 목록, 검색, 통계 등 6가지 명령어를 지원하는 CLI 기반 환자 관리 시스템으로, `util.py`에서 유틸리티 함수를 `import`해서 사용합니다. 프로그램 시작 시 기존 JSON 데이터를 자동으로 불러오고, 종료 시 자동 저장합니다. 입력값 검증(수축기 > 이완기)도 포함되어 있습니다.

**사용된 개념:** 모듈 import, while 루프, 명령어 분기(if/elif/else), CRUD 기능

### util.py — 유틸리티 함수 모듈

Day4_step3.py가 사용하는 공용 함수들을 모아둔 모듈입니다. 혈압판정, 환자브리핑, 저장하기(JSON 쓰기), 불러오기(JSON 읽기 + 에러 처리), 숫자입력(범위 검증 포함), 통계보기(평균 나이·진단별 분포·고혈압 비율) 등 6개 함수를 제공합니다. `try/except`를 사용한 에러 처리와 `os.path.exists`를 활용한 파일 존재 확인 로직이 포함되어 있습니다.

**사용된 개념:** 모듈화, json, os, try/except, 딕셔너리 집계, f-string 포맷팅
